💻 Lab 6: Race Results Manager

🎯 Overview

In this assignment you will build a race results manager that sorts race times using six different sorting algorithms and then analyzes the sorted data. You will implement every sort from CT12 (bubble, insertion, selection) and CT13 (merge, quick, heap) from scratch, plus utility functions that demonstrate why sorting matters.

  • O(n²) sorts — bubble sort (early exit), insertion sort (shift pattern), selection sort (find-min)
  • O(n log n) sorts — merge sort (divide and merge), quick sort (partition), heap sort (max-heap)
  • Race analysis — verify sorted order, find median time, remove duplicate times, merge two race heats

📖 Learning Objectives

By completing this assignment, you will practice:

  • All six sorting algorithms from CT12 and CT13 — implemented independently from memory
  • Recursive divide-and-conquer — merge sort's split/merge and quick sort's partition/recurse
  • Heap operations — building a max-heap and extracting elements in sorted order
  • Practical sorted-data operations — median, deduplication, and merging sorted arrays

💡 Key Concepts

O(n²) vs O(n log n)
The quadratic sorts (bubble, insertion, selection) use nested loops and slow down dramatically on large inputs. The efficient sorts (merge, quick, heap) use divide-and-conquer to handle large data. Both groups produce the same sorted result — the difference is speed.
Why the Race Analyzer?
Sorting isn’t an end goal — it’s a tool. Once race times are sorted, you can instantly find the median, remove duplicates by scanning adjacent elements, and merge two heats with a single pass. These operations would be much harder on unsorted data.
Common Pitfall: Recursion Base Cases
Merge sort and quick sort are recursive. If you forget the base case (left >= right or low >= high), you’ll get infinite recursion and a stack overflow crash. Always check the base case first.

🔧 Troubleshooting

CMake does not configure automatically

  1. Make sure you have Desktop development with C++ workload installed in Visual Studio
  2. Go to Project → CMake Settings and ensure the configuration is correct
  3. Try Project → Delete Cache and Reconfigure
  4. Verify that CMakeLists.txt is in the root of your cloned repository

Tests fail to build

  • Google Test is fetched automatically via CMake’s FetchContent — make sure you have an internet connection during the first build
  • Do not modify header files or test files

Running Tests

  1. In Visual Studio, select run_tests.exe as the startup item from the dropdown
  2. Press Ctrl+F5 to run without debugging
  3. Check the output for passed/failed test results
